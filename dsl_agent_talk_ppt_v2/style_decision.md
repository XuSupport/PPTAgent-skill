# Style Decision

## Selected Option
- Option C / Narrative Compiler

## Option Summary

### Option A
- 咨询架构风格
- 适合正式汇报、架构评审、管理层沟通
- 特点是结构稳定、信息分区清楚、最容易扩成整套方法论 deck

### Option B
- 工程蓝图风格
- 适合技术分享、团队内训、工程复盘
- 特点是像系统设计图，模块边界清楚，理论与实践并列呈现

### Option C
- 叙事编译器风格
- 适合大会分享、开场页、对外表达
- 特点是主张更强、记忆点更高，但整体气质更偏演讲型

## What To Keep
- 强主张开场
- 深色底 + 克制高亮
- 卡片化理论锚点
- “动态编译器”这一收束叙事

## What To Remove
- 过多装饰噪音
- 抢主体的背景元素
- 纯海报式炫技
