# Brief

## Topic
强约束 DSL 智能体搭建经验

## Audience
- 智能体平台建设者、AI 应用工程师、规则/查询/策略类 DSL 系统负责人、技术管理者

## Goal
- 基于现有大纲，做一套可用于分享或汇报的 PPT，讲清为什么强约束 DSL 智能体难、为什么单轮生成不够、以及如何用分层架构、确定性代码和闭环修复把系统做成“可交付”的工程系统

## Slide Count
10-11

## Aspect Ratio
16:9

## Language
zh

## Style Keywords
- 专业、工程化、清晰、有一点系统设计感、适合技术分享与管理层沟通

## Style Red Flags
- 避免安全海报风
- 避免过多背景噪音
- 避免堆满长段文字
- 避免纯学术论文页的拥挤感
- 避免花哨但不解释架构的视觉效果

## Constraints
- 页数以原始 Markdown 大纲的一级主题为主线，控制在 10-11 页
- 必须结合网页资料补充理论支撑，而不是只复述经验
- 理论补充优先来自可靠来源与原始论文/官方文档
- 先做 3 个风格候选页，待用户确认风格后再开始整套生成
- 最终内容要兼顾“能讲”和“能投屏”

## Core Narrative
1. 这类系统不是“懂语法的聊天助手”，而是把自然语言稳定编译成可执行 DSL 的工程系统
2. 真正能落地的关键不在 Prompt 花活，而在剪枝、分层、边界、校验与修复闭环
3. 最终目标不是首轮命中，而是最终可交付；本质更像动态编译器，而不是聊天机器人
