# Style Draft

## Theme Positioning
- 这套 PPT 要把“强约束 DSL 智能体搭建经验”讲成一套可复制的工程方法，而不是零散的实战心得

## Audience and Use Scene
- 用于技术分享、架构评审、内部布道、团队方法论沉淀，也适合给管理层解释为什么“直接上大模型”不够

## Desired Tone
- 工程化、可信、克制、带一点架构设计感；既能让技术人认可，也能让非一线实现者看懂

## Desired Visual Cues
- 清晰的流水线
- 模块边界明确
- 理论锚点少而准
- 关键结论可以一眼抓住
- 有系统图气质，但不显乱

## Must Avoid
- 不要像通用 AI 演示模板
- 不要做成赛博安全告警海报
- 不要背景大字压过正文
- 不要所有页面都用同一种复杂流程图
- 不要只讲抽象理念，不落到工程动作

## Representative Sample Page
- 用“封面 + 核心架构总览”作为候选页
- 页面要同时体现：
  - 主题主张：不是聊天机器人，而是动态编译器
  - 方法主线：意图降维 -> 逻辑蓝图 -> 语法翻译 -> 静态整流 -> 动态自愈
  - 理论锚点：约束解码 / 迭代修复 / HITL
  - 适合后续扩展成整套 deck 的视觉语言
